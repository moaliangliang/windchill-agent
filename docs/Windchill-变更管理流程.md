# Windchill 变更管理操作流程

> 适用版本：Windchill PDMLink 11.x / 12.x
> 文档版本：v1.0

---

## 前置条件

- 拥有 Windchill 登录账号
- 拥有变更创建权限

---

## 变更流程概述

```
提交变更请求 → 审批 → 执行变更 → 审批 → 关闭
     (CR)       (审核)   (CO/ECN)   (验证)
```

---

## 操作步骤

### 步骤 1：创建变更请求（CR）

1. 导航到 **"变更" → "变更请求"**
2. 点击 **"新建变更请求"（New Change Request）**
3. 填写主要字段：

| 字段 | 说明 |
|------|------|
| 编号 | 自动生成 |
| 主题 | 简要描述变更内容 |
| 紧急程度 | 紧急/高/中/低 |
| 描述 | 详细描述变更原因和内容 |
| 受影响对象 | 选择受影响的部件/文档 |

4. 点击 **"完成"**

### 步骤 2：变更审批

1. 系统自动启动审批工作流
2. 审批人收到任务通知
3. 审批人审查变更内容 → 批准/拒绝/退回
4. 审批通过后进入变更执行阶段

### 步骤 3：创建变更通告（CO/ECN）

1. 在 CR 详情页 → 点击 **"新建变更通告"（New Change Notice）**
2. 在变更通告中添加具体的变更操作：

| 操作类型 | 说明 |
|---------|------|
| 修改（Modify） | 修改现有部件属性 |
| 添加（Add） | 添加新部件到 BOM |
| 移除（Remove） | 从 BOM 中移除部件 |
| 取代（Replace） | 用新件替换旧件 |
| 废止（Obsolete） | 标记部件为废止 |

### 步骤 4：执行变更

1. 在变更通告中逐项执行变更操作
2. 每项操作完成后标记为"已完成"
3. 验证变更结果

### 步骤 5：关闭变更

1. 所有变更操作执行完毕后提交审核
2. 审核通过后变更通告自动关闭
3. 相关部件状态更新

---

## 常见问题

### Q1：变更被拒绝了怎么办？

查看拒绝原因 → 修改后重新提交。重大变更建议先沟通再提交。

### Q2：如何查看变更历史？

在变更单详情页 → **"审计"（Audit）** 标签页 → 查看所有操作记录。
